# 02-design.md

## 実装アプローチ

### 共通要素マッピング方式によるスクロール同期

左右のDiffPanelに表示されたHTMLのDOMを走査し、共通するブロック要素同士をマッチングする。一方のパネルをスクロールしたとき、表示領域上端に最も近いマッチ要素を特定し、もう一方のパネルの対応要素が同じ位置に来るようスクロールする。

### スクロール同期アルゴリズム

#### Step 1: ブロック要素の抽出

左右のDOM内のブロック要素（`h1`〜`h6`, `p`, `table`, `ul`, `ol`, `blockquote`, `pre`）を出現順に抽出し、各要素の `textContent` を取得する。

#### Step 2: LCS（最長共通部分列）ベースのマッチング

単純なインデックス対応ではなく、`textContent` を使ったLCSアルゴリズムで左右の要素をマッチングする。これにより、ブロック要素の追加・削除があっても正しく対応する要素同士をペアリングできる。

**マッチング手順:**

1. 左右のブロック要素からそれぞれ `textContent` のリストを作成
2. テキストの類似度判定でLCSを算出し、共通する要素ペアを特定
3. LCSでマッチしなかった要素（一方にしか存在しない追加・削除ブロック）はスキップ

**類似度判定:**

- 見出し（`h1`〜`h6`）: `textContent` の完全一致でマッチ（見出しはリビジョン間で安定しているため）
- その他のブロック要素（`p`, `table` 等）: `textContent` の前方一致（先頭20文字）でマッチ。編集による軽微な変更を吸収する

```
例:
左: [h1"はじめに", p"概要...", h2"設計", p"設計の..."]
右: [h1"はじめに", p"概要...", p"新段落", h2"設計", p"設計の..."]

LCSマッチ結果:
  h1"はじめに" ↔ h1"はじめに"    ← マッチ
  p"概要..."   ↔ p"概要..."      ← マッチ
  (右 p"新段落" はマッチなし)      ← スキップ
  h2"設計"     ↔ h2"設計"        ← 正しくマッチ!
  p"設計の..." ↔ p"設計の..."    ← 正しくマッチ!
```

#### Step 3: スクロール時の同期処理

1. スクロール元パネルの表示領域上端に最も近い（上端以上にある最後の）マッチ要素を特定
2. そのマッチ要素と次のマッチ要素の間でのスクロール進捗率を算出
3. もう一方のパネルの対応要素ペア間で同じ進捗率の位置にスクロール

### 補間（Interpolation）

マッチ要素間のスクロール位置は線形補間する。

```
例: 左パネルで要素Aと要素Bの間を50%スクロールした場合、
    右パネルでも対応する要素A'と要素B'の間を50%の位置にスクロールする。
```

これにより、マッチ要素間の区間でもスムーズに同期する。

## 変更するコンポーネント

### 1. `DiffPanel.tsx` - 変更

- `ref`（またはコールバックref）を外部に公開し、スクロールコンテナのDOM参照を取得可能にする
- `forwardRef` でラップするか、`containerRef` propsを受け取る形にする

### 2. `DiffModal.tsx` - 変更

- スクロール同期のオン/オフ状態を管理する `syncScroll` state を追加
- トグルUIをヘッダーまたはパネル上部に配置
- 左右のDiffPanelのスクロールコンテナrefを保持
- スクロール同期ロジック（カスタムフック `useSyncScroll`）を呼び出す

### 3. `useSyncScroll.ts` - 新規作成

スクロール同期のカスタムフック。以下の責務を持つ:

- 左右のスクロールコンテナのrefを受け取る
- `enabled` フラグでオン/オフを制御
- マッチ要素リストの構築（HTMLが変わるたびに再構築）
- `scroll` イベントリスナーの登録・解除
- 相互トリガー防止のロック機構
- スロットリング（`requestAnimationFrame` ベース）

```typescript
interface UseSyncScrollOptions {
  leftRef: RefObject<HTMLDivElement>;
  rightRef: RefObject<HTMLDivElement>;
  enabled: boolean;
  leftHtml: string;  // 変更検知用
  rightHtml: string; // 変更検知用
}
```

### 4. `syncScrollUtils.ts` - 新規作成

純粋関数群を分離:

- `extractBlockElements(container: HTMLElement): { element: HTMLElement; text: string; tag: string }[]` - ブロック要素とテキストを抽出
- `buildAnchorPairs(leftBlocks, rightBlocks): [HTMLElement, HTMLElement][]` - LCSベースでマッチングペアを構築。見出しは完全一致、その他は前方一致（先頭20文字）で類似度判定
- `computeTargetScroll(sourceContainer, targetContainer, anchorPairs, sourceSide): number` - 補間ベースのスクロール位置算出

## トグルUIの配置

モーダルヘッダーの「差分比較」タイトルと閉じるボタンの間に、スクロール同期トグルを配置する。

```
┌──────────────────────────────────────────────────────┐
│  差分比較          [🔗 スクロール連動] ☑        [×]  │
├──────────────────────────────────────────────────────┤
```

Bootstrapの `form-check form-switch` を使用したトグルスイッチとする。

## 相互トリガー防止

スクロールイベントが一方のパネルから発火し、もう一方をプログラム的にスクロールすると再度イベントが発火して無限ループになる。これを防ぐため:

- `isScrolling` フラグ（`useRef<boolean>`）を使用
- プログラム的スクロール開始時にフラグをON
- `requestAnimationFrame` の次フレームでフラグをOFF
- フラグがONの間はイベントハンドラをスキップ
