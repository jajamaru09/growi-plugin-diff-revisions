# 01-requirements.md - リビジョンナビゲーションボタン追加

## 変更・追加する機能の説明

差分比較モーダルにリビジョン番号を増減するボタンを追加し、UXを向上させる。

### 1. 左右パネル個別の増減ボタン
- 各ドロップダウンの左右に `◀` `▶` ボタンを配置
- クリックでそのパネルのリビジョン#を1つ増減する

### 2. 中央の同時増減ボタン
- 左右パネルの間（中央付近）に `◀` `▶` ボタンを配置
- クリックで左右両方のリビジョン#を同時に1つ増減する
- 例: 左=#3, 右=#4 の状態で `▶` → 左=#4, 右=#5

## ユーザーストーリー

- ユーザーとして、ボタンクリックで前後のリビジョンに素早く切り替えて差分を確認したい
- ユーザーとして、中央ボタンで左右同時にスライドさせて、連続した変更を追いかけたい

## 受け入れ条件

- [ ] 各ドロップダウンの横に ◀（前へ）▶（次へ）ボタンがある
- [ ] ボタンクリックでリビジョン#が1つ増減し、表示が更新される
- [ ] 最小(#1)または最大で境界を超えない（ボタン無効化）
- [ ] 中央に左右同時増減ボタンがある
- [ ] 中央ボタンで左右のリビジョン#が同時に増減する
- [ ] 中央ボタンはどちらかが境界に達したら無効化される

## 制約事項

- 既存のドロップダウン選択機能は維持する
- リビジョンNoの採番ロジックは変更しない
