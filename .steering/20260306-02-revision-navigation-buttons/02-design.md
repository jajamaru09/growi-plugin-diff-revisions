# 02-design.md - 実装設計

## 実装アプローチ

RevisionSelectorに増減ボタンを統合し、DiffModalに中央の同時増減ボタンを追加する。

## 画面設計（更新後ワイヤフレーム）

```
┌──────────────────────────────────────────────────────────────────┐
│  差分比較                                                   [×]  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────┐            ┌─────────────────────────┐
│  │ ▼ ドロップダウン [◀][▶] │  [◀][▶]    │ ▼ ドロップダウン [◀][▶] │
│  ├─────────────────────────┤            ├─────────────────────────┤
│  │                         │            │                         │
│  │  HTML表示領域           │            │  HTML表示領域           │
│  │                         │            │                         │
│  └─────────────────────────┘            └─────────────────────────┘
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
              個別ボタン(右寄せ)   中央ボタン    個別ボタン(右寄せ)
```

**個別ボタン配置**: ドロップダウンの右側に `◀` `▶` を隣接配置（マウス移動量を最小化）

## 変更するコンポーネント

### 1. `src/components/RevisionSelector.tsx`

- ドロップダウンの右側に `◀` `▶` ボタンを隣接配置（マウス移動量最小化）
- 新しいprops:
  - `onPrev: () => void` — リビジョン#を1つ減らす
  - `onNext: () => void` — リビジョン#を1つ増やす
  - `canPrev: boolean` — ◀ボタンの有効/無効
  - `canNext: boolean` — ▶ボタンの有効/無効
- レイアウト: `d-flex` で `[select] [◀][▶]` を横並び

### 2. `src/components/DiffModal.tsx`

- リビジョンIDからインデックスを引く関数を追加
- 各パネルのprev/next操作関数を定義
- 中央の同時増減ボタンUIを追加
- 中央ボタンの有効/無効判定: 左右どちらかが境界に達したら無効化

### ヘルパーロジック（DiffModal内）

```typescript
// revisionId → revisions配列のインデックスを取得
function getIndex(id: string): number

// 個別パネルの増減
function shiftRevision(currentId: string, delta: number): string

// 中央ボタンの増減（左右同時）
function shiftBoth(delta: number): void
```
