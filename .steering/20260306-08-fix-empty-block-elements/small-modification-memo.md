# small-modification-memo.md

## 要求（01-requirements 相当）

HTML比較モードで、`<ins>`/`<del>` タグを除去した際に、テキストが空になったブロック要素（`<li>`, `<p>`, `<h1>`-`<h6>` など）が反対側のパネルに空の要素として残ってしまうバグを修正する。

**再現条件:** 片方のリビジョンにのみ存在するリスト項目やパラグラフがある場合、htmldiff-js が `<ins>`/`<del>` でラップし、除去後に空のコンテナ要素が残る。

## 設計（02-design 相当）

**変更箇所:** `src/diffEngine.ts` の `computeDiff` 関数

**修正方針:** `<ins>`/`<del>` タグ除去後に、テキスト内容が空になったブロック要素を再帰的に除去するポストプロセスを追加する。

- 対象ブロック要素: `li`, `p`, `h[1-6]`, `tr`, `td`, `th`, `blockquote`, `pre`, `div`
- 空の判定: タグ除去後に空白文字のみ、または子要素がすべて空の場合
- 繰り返し適用: 内側の要素を除去した結果、外側も空になるケース（例: `<ul>` 内の `<li>` がすべて空 → `<ul>` も除去）に対応
