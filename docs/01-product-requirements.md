# プロダクト要求定義書

## 1. プロダクトビジョンと目的

GROWIのWiki記事における過去のリビジョン間の差分を、HTMLレンダリング形式で視覚的に比較できるプラグインを提供する。
ユーザーが記事の変更履歴を効率的に確認・理解できるようにすることで、チームのナレッジマネジメントを強化する。

## 2. ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
- GROWIを利用するチーム・組織のメンバー
- 記事の編集履歴を確認する必要があるユーザー（レビュアー、管理者等）

### 課題・ニーズ
- 現状のGROWIでは、リビジョン間の差分をMarkdownソースレベルでしか確認できず、実際の表示上の変更を把握しにくい
- HTML表示形式での差分比較により、最終的な見た目ベースで変更内容を直感的に把握したい
- 2つの任意のリビジョンを選択して比較したい

## 3. 主要な機能一覧

| # | 機能名 | 概要 |
|---|--------|------|
| F-01 | 差分比較モーダル起動ボタン | サイドバーに差分比較モーダルを開くボタンを配置 |
| F-02 | リビジョン選択 | ドロップダウンから比較対象の2つのリビジョンを選択 |
| F-03 | HTMLレンダリング表示 | 選択したリビジョンのMarkdownをGROWIと同等のHTMLにレンダリングして表示 |
| F-04 | 差分ハイライト | 2つのHTMLの差分部分を色分け（追加：淡い緑、削除：淡いピンク）で強調表示 |
| F-05 | GROWIと同一のCSS適用 | モーダル内のHTMLがGROWIの記事閲覧画面と同じ見た目になるようCSSを適用 |

## 4. 成功の定義

- ユーザーが任意の2リビジョンを選択し、HTML表示形式で差分を確認できる
- モーダル内のHTMLレンダリングがGROWIの記事閲覧画面と同等の見た目である
- 差分箇所が色分けで視覚的に識別できる
- 既存のGROWI機能に影響を与えない（プラグインとして独立動作）

## 5. ビジネス要件

- GROWIプラグインの仕様（スキーマv4、scriptタイプ）に準拠する
- OSSとして公開可能なライセンス（MIT）で開発する
- GROWIのバージョンアップに対して一定の互換性を保つ

## 6. ユーザーストーリー

### US-01: リビジョン差分の比較
> GROWIユーザーとして、閲覧中の記事ページでサイドバーのボタンをクリックし、差分比較モーダルを開きたい。モーダルでは2つのリビジョンをドロップダウンから選択し、左右のパネルにそれぞれのリビジョンのHTML表示を確認したい。差分箇所は色分けで強調されていてほしい。

### US-02: リビジョンの識別
> GROWIユーザーとして、ドロップダウンでリビジョンを選択する際に、リビジョン番号・リビジョンID・更新日時・更新者が表示されていることで、どのリビジョンかを容易に識別したい。

## 7. 受け入れ条件

- [ ] サイドバーに差分比較モーダルを開くボタンが表示される
- [ ] ボタンクリックでモーダルが開く
- [ ] モーダル内に左右2つの表示領域がある
- [ ] 各表示領域の上部にリビジョン選択ドロップダウンがある
- [ ] ドロップダウンに「リビジョンNo - リビジョンID - YYYY/MM/DD HH:mm 更新者」形式で表示される
- [ ] リビジョンNoは日付ソートで古い順に1から採番される
- [ ] 選択したリビジョンのMarkdownがHTMLにレンダリングされて表示される
- [ ] HTMLレンダリングはGROWIの記事閲覧画面と同等の見た目である
- [ ] 2つのリビジョン間の差分箇所が淡いピンク（削除）と淡い緑（追加）でハイライトされる
- [ ] モーダルを閉じることができる（Escキー、閉じるボタン、背景クリック）

## 8. 機能要件

### FR-01: プラグイン基盤
- GROWIプラグイン仕様（スキーマv4、scriptタイプ）に準拠
- `window.pluginActivators` によるactivate/deactivateライフサイクル管理
- Navigation APIによるページ遷移検知

### FR-02: サイドバーボタン
- GROWIサイドバーのボタン群に統合される形でボタンを配置
- ボタンのスタイルは既存ボタンと統一

### FR-03: リビジョン取得
- GROWI APIを使用して閲覧中ページの全リビジョン一覧を取得
- リビジョン情報（ID、body、更新日時）を取得

### FR-04: Markdownレンダリング
- GROWIと同等の手順でMarkdownをHTMLに変換
- GROWIのソースコードを解析し、同等のレンダリングパイプラインをプラグイン内に実装

### FR-05: CSS適用
- GROWIの記事閲覧画面と同じCSSをモーダル内のHTMLに適用
- 既存のGROWI CSSクラスを活用

### FR-06: 差分検出・ハイライト
- 2つのHTMLを比較し差分を検出
- 追加部分：淡い緑色の背景
- 削除部分：淡いピンク色の背景

### FR-07: モーダルUI
- 左右分割レイアウト
- 各パネル上部にリビジョン選択ドロップダウン
- ドロップダウン表示形式：「#No - revisionId - YYYY/MM/DD HH:mm 更新者」
- モーダル閉じ操作（Escキー、閉じるボタン、背景クリック）

## 9. 非機能要件

### NFR-01: パフォーマンス
- モーダル表示時のリビジョン一覧取得は3秒以内
- リビジョン選択後のHTML差分表示は5秒以内

### NFR-02: 互換性
- GROWIのプラグインシステム（スキーマv4）に準拠
- モダンブラウザ（Chrome、Firefox、Edge最新版）で動作

### NFR-03: 保守性
- TypeScriptによる型安全な実装
- コンポーネントベースのアーキテクチャ

### NFR-04: セキュリティ
- XSS対策（HTMLレンダリング時のサニタイズ）
- GROWI APIへのアクセスは既存の認証を利用
